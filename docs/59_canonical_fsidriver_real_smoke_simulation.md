# Step 59 Canonical FSIDriver Real Smoke Simulation

Step 59 verifies that the canonical `FSIDriver3D` implementation can run a real minimal simulation after the Step 58 migration.

## Scope

The exercised implementation owner is:

- `src/mpm_lbm/sim/drivers/fsi_driver.py`

The smoke matrix imports from:

- `src.mpm_lbm.sim.drivers.fsi_driver`

and calls:

```python
driver.run()
```

The required smoke rows are:

- `canonical_driver_none_16_1step`
- `canonical_driver_penalty_16_1step`
- `canonical_driver_moving_boundary_engineering_16_1step`

All required rows use `n_grid = 16`, `n_particles = 512`, `n_lbm_steps = 1`, `mpm_substeps_per_lbm_step = 1`, `geometry_type = box`, and lightweight output settings only.

## Geo Path Naming

Step 59 fixes the canonical driver geometry-output path so the filename follows the configured grid size:

```text
geo_all_fluid_{n_grid}.dat
```

The naming audit checks 16, 32, and 48 grid constructors and verifies constructor-only probing does not create output files.

## Output Boundary

Each Step 59 driver run directory may contain only:

- `driver_config.json`
- `geo_all_fluid_16.dat`
- `diagnostics_timeseries.csv`
- `diagnostics_timeseries.npz`

The output guard rejects VTR output, particle NPY output, large files, private absolute paths in Step 59 outputs/logs, edits under `external/taichi_LBM3D`, and edits under `data/real_geometry_candidates`.

## Non-Goals

Step 59 does not activate runtime geometry, moving-wall velocity, larger-grid validation, one-cycle validation, link-area production validation, real squid geometry, squid swimming, jet validation, grid convergence, tau migration, or production readiness.

Step 59 does not migrate the real boundary-motion, geometry-motion, wall-velocity, runtime-geometry, diagnostic-geometry, geometry-displacement, squid-proxy, real-geometry candidate, or external solver implementations.

## Interpretation

Passing Step 59 means the canonical driver can perform a real one-step smoke run for the existing `none`, `penalty`, and `moving_boundary` engineering modes at 16^3 with 512 particles, and that the generated artifacts satisfy strict lightweight-output and Step 58 regression guards.

Passing Step 59 does not mean the solver is validated for larger cases, moving squid-like geometry, real jet propulsion, real squid simulations, or production use.
