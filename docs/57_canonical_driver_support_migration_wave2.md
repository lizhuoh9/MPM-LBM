# Step 57 Canonical Driver Support Migration Wave 2

Step 57 migrates driver support, diagnostics, link-area support, run utilities, and geometry support implementations into the canonical package tree created by Step 55 and started by Step 56.

## Scope

The migrated implementation owners are:

- `src/mpm_lbm/sim/drivers/sim_config.py`
- `src/mpm_lbm/sim/drivers/fsi_config.py`
- `src/mpm_lbm/sim/io/run_utils.py`
- `src/mpm_lbm/diagnostics/fsi_diagnostics.py`
- `src/mpm_lbm/sim/coupling/link_area_accounting.py`
- `src/mpm_lbm/sim/coupling/link_area.py`
- `src/mpm_lbm/sim/coupling/momentum_accounting.py`
- `src/mpm_lbm/sim/geometry/config.py`
- `src/mpm_lbm/sim/geometry/utils.py`
- `src/mpm_lbm/sim/geometry/mesh_io.py`
- `src/mpm_lbm/sim/geometry/mesh_quality.py`
- `src/mpm_lbm/sim/geometry/voxel_io.py`
- `src/mpm_lbm/sim/geometry/voxel_quality.py`
- `src/mpm_lbm/sim/geometry/importers.py`
- `src/mpm_lbm/sim/geometry/sampler.py`
- `src/mpm_lbm/sim/geometry/quality.py`

The corresponding legacy root files are compatibility shims that import from those canonical modules.

## Dependency Direction

The accepted direction after this step is:

```text
legacy root import -> compatibility shim -> canonical implementation
```

The rejected direction is:

```text
canonical implementation -> legacy root implementation
```

## Non-Goals

Step 57 does not migrate `src/fsi_driver.py`, boundary-motion implementation modules, wall-velocity modules, squid modules, runtime-geometry modules, diagnostic-geometry modules, geometry displacement modules, real-geometry intake/candidate/fingerprint/normalization modules, `external/taichi_LBM3D`, or `data/real_geometry_candidates`.

Step 57 also does not add a new `48^3`/`64^3` run, longer cycle run, tau migration, historical physics formula change, default solver behavior change, real squid validation, jet validation, or production-readiness claim.

## Interpretation

Passing Step 57 means the Wave 2 support-layer ownership batch is now canonical-package owned, legacy imports remain compatible, lazy `src` exports point to canonical modules for migrated symbols, and lightweight behavior-preservation checks remain green.

Passing Step 57 does not mean real squid validation, jet validation, grid convergence, production readiness, or new FSI physics has been established.
