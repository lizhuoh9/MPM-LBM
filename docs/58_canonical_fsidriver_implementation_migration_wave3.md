# Step 58 Canonical FSIDriver Implementation Migration Wave 3

Step 58 migrates the real `FSIDriver3D` implementation from the legacy root file into the canonical driver package.

## Scope

The new implementation owner is:

- `src/mpm_lbm/sim/drivers/fsi_driver.py`

The legacy root file remains a compatibility shim:

- `src/fsi_driver.py`

The accepted dependency direction is:

```text
legacy root import -> compatibility shim -> canonical implementation
```

The rejected dependency direction is:

```text
canonical implementation -> legacy root implementation
```

## Temporary Bridge Surfaces

`FSIDriver3D` still references optional hook surfaces that have not been fully migrated. Step 58 therefore adds canonical bridge modules that deliberately forward to the legacy implementations until Step 59:

- `src/mpm_lbm/sim/motion/boundary_motion_interface.py`
- `src/mpm_lbm/sim/motion/geometry_motion_interface.py`
- `src/mpm_lbm/sim/wall_velocity/application.py`

These bridge modules are temporary import surfaces only. They are not real implementation migrations.

## Non-Goals

Step 58 does not migrate the real boundary-motion implementation, wall-velocity implementation, runtime geometry implementation, diagnostic geometry implementation, geometry-displacement implementation, squid modules, real-geometry candidate modules, `external/taichi_LBM3D`, or `data/real_geometry_candidates`.

Step 58 also does not add larger runs, one-cycle runtime geometry validation, new FSI physics, jet validation, real squid validation, grid convergence, tau migration, or production-readiness claims.

## Interpretation

Passing Step 58 means `FSIDriver3D` and `DIAGNOSTIC_FIELDS` are owned by the canonical driver module, old `src.fsi_driver` imports resolve through a thin shim, optional hook imports resolve through temporary bridge surfaces, and constructor-only behavior preservation checks remain green.

Passing Step 58 does not mean the optional hook implementations are migrated, runtime geometry is advanced, wall velocity behavior is expanded, real squid validation is established, or production readiness has been proven.
