# Step93 Taichi GGUI Visualization Enablement Plan And Guard

Step93 is a corrective plan-and-guard-only step for the future Step94 Taichi
GGUI visualization smoke. It does not instantiate `FSIDriver3D`, does not call
`driver.run()`, does not execute simulation, does not open a GGUI window, and
does not write screenshots.

Step93 replaces the previous Step93 file-output direction with exactly one
future Step94 row:

```text
first_user_squid_proxy_runtime_geometry_diagnostic_only_wall_velocity_solid_vel_32_1step_ggui_visual_smoke
```

The planned Step94 envelope is:

```text
n_grid = 32
n_particles = 1024
n_lbm_steps = 1
mpm_substeps_per_lbm_step = 1
coupling_mode = moving_boundary
reaction_transfer_mode = engineering
geometry_type = squid_proxy
geometry_motion_application_mode = diagnostic_only
wall_velocity_application_mode = solid_vel_experimental
target_lbm_field = solid_vel
target_u_lbm = [0.0, 0.0, 0.0]
ggui_visualization_enabled = true
write_vtk = false
write_particles = false
output_interval = 1
```

The only planned new feature from Step92 to Step94 is:

```text
taichi_ggui_visualization
```

The reduction from Step92's 10 LBM steps to Step94's 1 LBM step is intentional
GGUI visualization-path isolation, not a duration regression and not a new
physics claim.

## Guarded Boundaries

Step93 keeps these boundaries:

- no Step93 driver-run directory;
- no GGUI window;
- no screenshot output;
- no particle NPY output;
- no file-based visualization artifact output;
- no real geometry candidate data;
- no link-area transfer;
- no 48^3 or 64^3 row;
- no dense wall-velocity output;
- no dense displacement output;
- no solver formula change;
- no tau migration;
- no physical validation, real squid validation, squid swimming, squid actuation,
  visualization-readiness, or production-readiness claim.
