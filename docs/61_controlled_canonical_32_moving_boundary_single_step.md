# Step 61 Controlled Canonical 32 Moving-Boundary Single-Step

Step 61 expands the canonical real-driver evidence from the Step 60 16^3 duration ramp to a single 32^3 moving-boundary engineering probe.

## Scope

The exercised implementation owner remains:

- `src/mpm_lbm/sim/drivers/fsi_driver.py`

The matrix runner imports the canonical driver and calls:

```python
driver.run()
```

Required row:

- `canonical_driver_moving_boundary_engineering_32_1step`

The required row uses `n_grid = 32`, `n_particles = 1024`, `n_lbm_steps = 1`, `mpm_substeps_per_lbm_step = 1`, `geometry_type = box`, disabled VTK output, disabled particle output, disabled quality checks, static boundary motion, disabled wall-velocity application, static geometry motion, and disabled geometry-motion application.

Optional 32^3 penalty and 3-step configs are checked in but disabled by default.

## Output Boundary

The required Step 61 driver run directory may contain only:

- `driver_config.json`
- `geo_all_fluid_32.dat`
- `diagnostics_timeseries.csv`
- `diagnostics_timeseries.npz`

The output guard rejects VTR output, particle NPY output, large files, private absolute paths in Step 61 outputs/logs, edits under `external/taichi_LBM3D`, and edits under `data/real_geometry_candidates`.

## Non-Goals

Step 61 does not activate runtime geometry, wall-velocity application, geometry motion, prescribed boundary motion, 48^3 or 64^3 rows, required 3-step or 5-step rows, long-duration rows, required link-area rows, real geometry, propulsion validation, real squid behavior validation, grid-convergence proof, viscosity/tau migration, heavy visualization output, particle output, runtime solver code changes, or deployment-readiness claims.

## Interpretation

Passing Step 61 means the existing canonical driver can run a finite/bounded 32^3 single-step feasibility smoke for the moving-boundary engineering mode while preserving Step 60 and output-guard boundaries.

Passing Step 61 does not establish 32^3 duration behavior, runtime-geometry behavior, wall-velocity behavior, larger-grid behavior, or final physical correctness.
